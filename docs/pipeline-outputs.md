# Pipeline Outputs Reference

Catalog of files generated by the q100-variant-benchmark Snakemake pipeline and the
R loaders in `R/data_loading.R` that read them.

For column-level descriptions of each loaded data frame, see [data-dictionary.md](data-dictionary.md).

## Overview

Outputs are organized by **benchmark identifier** using the pattern
`{bench_version}_{ref}_{bench_type}` — e.g., `v5.0q_GRCh38_smvar`.

| Component | Values |
|---|---|
| `bench_version` | `v0.6`, `v4.2.1`, `v5.0q` |
| `ref` | `GRCh37`, `GRCh38`, `CHM13v2.0` |
| `bench_type` | `smvar` (small variants <50 bp), `stvar` (structural variants ≥50 bp) |

Outputs fall into two tiers:

- **Aggregated metrics** (KB): pre-computed counts/percentages, fast to load
- **Detailed data** (MB–GB): per-variant Parquet, per-base coverage BEDs

All loading functions support `use_cache = TRUE` (default) and `force_refresh = FALSE`.
Cache files live in `analysis/cache/` (Parquet + zstd, invalidated by source mtime).

## Output Directory Layout

```
results/
├── genomic_context/{benchmark}/
│   ├── coverage/{context}_cov.bed              # bedtools coverage output (large)
│   ├── genomic_context_coverage_table.csv      # per-context overlap metrics (small)
│   └── variants_by_genomic_context.parquet     # per-context variant counts (small)
├── variant_tables/{benchmark}/
│   └── variants.parquet                        # full variant-level data (~1 GB)
├── exclusions/{benchmark}/                     # v5.0q only
│   ├── exclusion_impact.csv                    # Q1: per-exclusion BED + variant counts
│   └── exclusion_interactions.csv              # Q2: upset-style combination overlaps
├── exclusions/{comp_id}/                       # v5.0q cross-version
│   └── old_only_*.csv                          # Q3: regions/variants in v4.2.1/v0.6 only
├── ref_genome_sizes/{ref}_size.tsv             # samtools faidx output
└── chr8_synteny/                               # multi-panel SyRI/plotsr figure
    ├── syri/{ref_samp}_{qry_samp}syri.out
    ├── inversion_coords.json
    └── chr8_figure.{pdf,png}
```

Coverage BEDs and variant Parquet are gitignored — generated by `make run`.

## Aggregated Metrics

### `variants_by_genomic_context.parquet`

**Path:** `results/genomic_context/{benchmark}/variants_by_genomic_context.parquet`
**Generator:** `count_variants_by_genomic_context` rule
**Loader:** `load_genomic_context_metrics()`

Long-format counts by `(context_name, var_type, szbin)`. The R loader pivots
variant types into wide columns (`snv_count`, `indel_count`, `del_count`, `ins_count`,
`complex_count`) and joins with `genomic_context_coverage_table.csv` to produce a
single tibble with coverage metrics + counts. See data-dictionary for full schema.

Raw schema (4 columns): `context_name`, `var_type`, `szbin`, `count`.

```r
metrics <- load_genomic_context_metrics()                  # all benchmarks
metrics <- load_genomic_context_metrics(
  benchmark_filter = c("v5.0q_GRCh38_smvar")               # subset
)
```

### `genomic_context_coverage_table.csv`

**Path:** `results/genomic_context/{benchmark}/genomic_context_coverage_table.csv`
**Generator:** `compute_genomic_context_coverage_table` rule (uses `compute_coverage_table.py`)

Per-context BED overlap with the benchmark regions:
`context_name`, `context_bp`, `intersect_bp`, `pct_of_context`, `pct_of_bench`.

Joined into the data frame returned by `load_genomic_context_metrics()`; not loaded
separately.

### `exclusion_impact.csv` (v5.0q only)

**Path:** `results/exclusions/{benchmark}/exclusion_impact.csv`
**Generator:** `compute_exclusion_impact` rule
**Loader:** `load_exclusion_metrics()`

Per-exclusion impact on the dipcall regions: BED overlap stats + variant counts
(snv/indel for smvar, del/ins for stvar). Source columns `exclusion`,
`dip_intersect_bp`, `snp_count`, `sv_ins_count`, `sv_del_count` are renamed by the
loader. See data-dictionary for full schema.

### `exclusion_interactions.csv` (v5.0q only)

**Path:** `results/exclusions/{benchmark}/exclusion_interactions.csv`
**Generator:** `compute_exclusion_interactions` rule
**Loader:** `load_exclusion_interactions()`

Upset-style decomposition: which exclusion combinations overlap which bases/variants.
Useful for Q2 ("which exclusions are redundant?"). Schema in data-dictionary.

### Cross-version exclusion files (v5.0q comparisons)

**Path:** `results/exclusions/{comp_id}/old_only_*.csv`
**Generator:** `annotate_old_benchmark_status` rule

Identifies regions/variants present in older benchmarks (v4.2.1, v0.6) but excluded
from v5.0q — used for Q3 ("what did we lose?"). Loaded ad hoc in
`benchmark_unique_regions.qmd`; no dedicated loader.

### `ref_genome_sizes/{ref}_size.tsv`

**Generator:** `samtools faidx` + `seqkit stats`
**Loader:** `load_reference_sizes()`

Per-chromosome length, ATGC bases, N bases. The R loader adds `asm_bp = length - ns`.

## Detailed Data

### `variants.parquet`

**Path:** `results/variant_tables/{benchmark}/variants.parquet`
**Generator:** `generate_variant_parquet` rule (uses Truvari `VariantRecord` API)
**Loader:** `load_variant_table()`, `load_all_variant_tables()`

Per-variant data with INFO/CONTEXT_IDS and INFO/REGION_IDS annotations. Schema is
defined in `R/schemas.R::get_arrow_schema("variant_table")` — see data-dictionary
for the column list.

⚠️ Large (~500 MB – 2 GB per benchmark). Always filter:

```r
variants <- load_variant_table(
  "v5.0q_GRCh38_smvar",
  filters = list(
    chromosomes = c("chr1"),
    variant_types = c("SNP", "INS"),     # use Truvari names: SNP, DEL, INS, DUP, INV, BND
    in_benchmark_only = TRUE             # default
  )
)
```

`load_all_variant_tables()` iterates all benchmarks; same filter API.

### `genomic_context/{benchmark}/coverage/{context}_cov.bed`

**Generator:** `genomic_context_coverage` rule (`bedtools coverage`)
**Loader:** `load_genomic_context_coverage()`

Per-interval coverage of difficult-region BEDs by the benchmark. Schema in data-dictionary.

```r
coverage <- load_genomic_context_coverage("v5.0q_GRCh38_smvar")
hp_tr     <- load_genomic_context_coverage(
  "v5.0q_GRCh38_smvar",
  context_filter = c("HP", "TR")
)
```

### Benchmark + Platinum Pedigree region BEDs

**Loaders:** `load_benchmark_regions()`, `load_platinum_pedigree_regions()`

Standardized interval tibbles (`chrom`, `start`, `end`, `interval_size` + benchmark
metadata). Both honor caching.

## Loading Functions Summary

| Function | Reads | Returns |
|---|---|---|
| `load_genomic_context_metrics()` | `genomic_context_coverage_table.csv` + `variants_by_genomic_context.parquet` | tibble: per-context coverage + variant counts |
| `load_exclusion_metrics()` | `exclusion_impact.csv` (v5.0q only) | tibble: per-exclusion BED + variant counts |
| `load_exclusion_interactions()` | `exclusion_interactions.csv` (v5.0q only) | tibble: upset-style combinations |
| `load_reference_sizes()` | `ref_genome_sizes/*_size.tsv` | tibble: per-chromosome sizes |
| `load_hg002q100_size()` | HG002 Q100 maternal `.fai` (downloaded) | numeric: total assembly bp |
| `load_benchmark_regions()` | benchmark BEDs in `resources/` | tibble: intervals with metadata (cached) |
| `load_platinum_pedigree_regions()` | Platinum Pedigree BEDs | tibble: intervals (cached) |
| `load_variant_table()` | `variants.parquet` (one benchmark) | tibble: per-variant rows (cached) |
| `load_all_variant_tables()` | all `variants.parquet` files | tibble: combined per-variant rows |
| `load_genomic_context_coverage()` | `coverage/*_cov.bed` files | tibble: per-interval coverage (cached) |
| `load_primary_analysis_data()` | bundles the above | named list (see data-dictionary) |

All loaders accept `use_cache` and `force_refresh`. Cache utilities:

```r
cache_info()                           # list cached datasets
invalidate_cache("variant_table")      # remove one
clear_cache()                          # remove all
```

## Recommended Workflow

```r
source(here::here("R/data_loading.R"))

# Most analyses: aggregated metrics + reference sizes
metrics    <- load_genomic_context_metrics()
ref_sizes  <- load_reference_sizes()

# Bundle everything used by the primary notebooks
data <- load_primary_analysis_data()
data$genomic_context_metrics_df
data$variants_df
data$ref_sizes_df

# Variant-level investigations: always filter
variants <- load_variant_table(
  "v5.0q_GRCh38_smvar",
  filters = list(chromosomes = c("chr1"), variant_types = c("INS"))
)
```

## Troubleshooting

**No `variants_by_genomic_context.parquet` files found** — run `make run` (or
`snakemake --cores 14 --sdm conda`); these are gitignored.

**Slow first load of variant tables** — Parquet cache populates on first read;
subsequent reads use the cache. Pass `force_refresh = TRUE` to rebuild.

**Missing exclusion files** — `load_exclusion_metrics()` returns an empty tibble
with a warning when no v5.0q outputs exist. Confirm with
`fs::dir_ls("results/exclusions", recurse = TRUE, glob = "**/exclusion_impact.csv")`.

**Chromosome name mismatches** — the loaders standardize via `std_chrom()`. If you
load BED files manually, apply the same helper.
